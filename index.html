Put it all together

Daily workflow

This is what a typical dev session looks like using everything you've set up.

# 1. Open WezTerm # 2. Start or attach tmux session tmux new -s work # first time tmux attach # coming back # 3. Split your workspace Ctrl+a v # split vertically Ctrl+a s # split horizontally # 4. Open Neovim in main pane nvim . # 5. Use second pane for terminal # run builds, git commands, etc. clang++ main.cpp -o main && ./main # 6. Git with lazygit lg # 7. Browse files with yazi y # 8. When done — detach, not close Ctrl+a d # everything keeps running # 9. Come back tomorrow tmux attach # right where you left off
Pro tip

Never close tmux

Always detach with Ctrl+a d. Your session stays alive, servers keep running, files stay open.

Pro tip

Name your sessions

tmux new -s projectname keeps things organized when you have multiple projects running.

Pro tip

Harpoon your files

Space H in Neovim to add a file to harpoon. Space h opens the quick menu. Space 1-9 to jump instantly.

Pro tip

Learn motions not arrows

The faster you stop using h/j/k/l and start using w/b/f/t/s, the faster you'll move in Neovim.

Pro tip

Space ff over :e

Use Space ff to open files instead of typing :e path/to/file. Fuzzy find is always faster.

Pro tip

z before cd

Once zoxide learns your directories, you'll never type a full path again. Just z proj and you're there.
